* IP, MAC 관련 내용

IT에서 주로 Physical하다 또는 logical하다 라는 말을 자주 쓰는데 바뀔 수 있는가의 유무로 보면 된다.

PC에 물리적인[Physical/바꿀수x] MAC 주소(Media Access Control)와 logical한[바꿀수o] IP 주소가 부여되어 있어야 통신을 할 수 有.

IP는 크게 v4와 v6으로 구분할 수 있고, IP 개념이 나온지 오래되었기에 주소 부족현상이 발생하여 이로 인해 v6이 탄생하게 되었다.

→ v6 주소체계를 현재 많이 쓰이고 있지는 않지만 (대부분 v4로), IOT 시장이 활성화되면 모든 IOT 기기에 IP 주소를 부여해야 하므로 사용해야 하는 시기가 올 것.

v4는 4개의 옥텟 구조로 되어 있고 → 8bit,8bit,8bit,8bit / 8비트가 최대로 표현할 수 있는 10진수 정수범위는 0~255이다.

v4 주소체계에서 예약된 주소가 있는데 아래와 같다.

0.0.0.0 - IP할당이 안되었다는 의미
255.255.255.255 - BroadCast 주소 [한 네트워크에 연결된 모든 pc에 전부 뿌리는 방식] / 대표적으로 통신방식에 Unicast(1:1), BroadCast(1:All)가 있다.
127.0.0.1 - localHost [자신과 통신할 때 쓰는 IP]

IP를 구성하는 방법에는 크게 2가지로 Static[정적] 구성, Dynamic[동적] 구성이 有.

Static 구성은 직접 IP를 입력하는 것이고, 고정된 IP를 사용하는 것. → 서버, 프린터와 같이 서비스를 제공하는 장치에 할당할 때 사용.
Dynamic 구성은 DHCP 프로토콜을 이용하여 DHCP 서버로부터 IP를 할당받는 것. / 고정되지 않는 유동적으로 변경될 수 有. → 주로 DHCP는 Clint와 같이 고정된 주소가 필요하지 않을 때 사용.

IP 주소에는 위에 언급한 예약된 주소 이외에 공인 IP와 사설 IP로 구분할 수 있다.

1) 공인 IP - 인터넷 사용가능 주소 → ISP 업체[SK,KT,LG]에 비용지불 후 공인 IP를 제공받는다
2) 사설 IP - 인터넷 사용X
- 사설 IP 대역은 10.x.x.x / 172.16.x.x ~ 172.31.x.x / 192.168.x.x 등

→ 공인 IP는 Unique하지만 사설 IP는 서로 다른 네트워크 망이라면 중복되도 상관X
→ 공인 IP 주소를 효율적으로 사용할 수 있고 보안성도 좋다. (NAT - Network Address Translation / 네트워크 주소 변환) / NAT 기술은 수료할때까지 나오는 기술
→ 일반 가정 내에서 사용하는 공유기 포트에 연결해서 여러 대의 PC를 사용한다 가정했을 때, 각각의 PC에 사설 IP를 부여하고 외부로 나가는 IP는 모든 PC가 공인 IP 1개로 동일
    But 그럼 데이터를 전송받을시 어떻게 구분하느냐 각각의 PC의 프로토콜 스택이 임의의 포트를 선택하기에 포트번호를 통해 구분되어 보내진다.

클라우드에서 "터널링"이라는 기술을 많이 듣게 될텐데, 물리적으로 엄청 떨어져 있거나 중간에 큰 장애물이 있더라도 "터널"을 뚫어 매우 빠르게 통신하게 할 수 있다
