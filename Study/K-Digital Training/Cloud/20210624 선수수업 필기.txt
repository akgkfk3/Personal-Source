* 브라우저에서 URL 검색시 일어나는 과정들 간단 설명

클라이언트 - 웹 브라우저에서 URL 검색시 도메인 부분에 해당하는 IP를 조회하기 위해 DNS 서버에 접속후 조회하고 클라이언트 랜 카드를 거쳐 서버쪽으로 전송한다

서버쪽 컴퓨터도 똑같이 물리적 자원(CPU,RAM,..)등을 관리하기 위한 운영체제가 있고 주로 Linux 사용됨.

클라이언트에서 전송된 정보를 서버 OS가 올바른 서비스[포트번호]로 보내준다 (윈도우-서비스, 리눅스-데몬 / OS와 함께 시작하여 특정 기능을 수행하기 위해 백그라운드에 존재하는 프로세스)

→ 어제 리눅스 서버 설치하며 했었던 httpd가 이러한 데몬이고, 비슷한 서비스로 apache, nginx, IIS 등이 有.

해당 서비스에서 클라이언트가 요구한 URL의 페이지[HTML,CSS,JS]파일등을 박싱해서 클라이언트쪽으로 다시 전송하게 된다.

DB에 접속해서 얻을 정보가 있다면 웹 서버에서 다시 DB서버로 요청을 하게 된다. (주로 3-Tier 계층으로 이루어짐)

∴ 우리 과정에서 배울 웹 언어는 pHp이고 DB로는 Mysql, MariaDB를 배우게 될 것이다.
